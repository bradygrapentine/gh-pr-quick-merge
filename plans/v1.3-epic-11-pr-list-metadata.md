## Goals

- Give the `/pulls` list at-a-glance signal density without hurting first-paint or row-injection p95.
- Per-row badges: CI status, PR size, comments count + jump-to-comments.
- Above-list quick-filter chips: **Mine / Ready / Stale / Small**, plus noise toggles for **Dependabot / Drafts**.
- Ready-to-merge rows visually highlighted (subtle row tint) so a green "go" jumps off the page.
- All client-side; no new endpoints beyond what GitHub already exposes.

## Non-goals

- Per-row CI rerun action (defer; risk + scope creep).
- Customizable filter chips beyond the four shipped (defer to v1.4 if user demand surfaces).
- Cross-repo "global mine" view (the popup already does this via `/search/issues`).
- GitLab parity — Epic 9 will absorb the badge contracts when adapters land.

## Story breakdown

### F11.1 — Row metadata badges (Track A)

#### QM-500 — CI status badge
- **Files:** `lib/qm-row-badges.js`, `theme.css`, `lib/hosts/github/pr-state.js` (extend cached state with `combined_status` summary), `content.js` (one call site).
- **Behavior:** combined-status fetch returns `{ state: "success" | "failure" | "pending" | null }`. Render a colored dot (`--qm-success` / `--qm-danger` / `--qm-warn` / hidden when null). Tooltip on hover shows the failing check name when `failure`.
- **Tests:** unit — classifier; integration — DOM mounting; visual — green / red / yellow variants.
- **AC:** badge mounts inside the existing `.qm-container` row container; injection idempotent (already gated by `INJECTED_ATTR`).

#### QM-501 — PR size badge
- **Files:** `lib/qm-row-badges.js`, `lib/qm-size-classify.js` (new pure-fn module), `theme.css`.
- **Behavior:** classify `(additions + deletions)` into XS (<10), S (<50), M (<200), L (<500), XL (≥500). Mirrors GitHub's PR-size convention. Badge shows `S · 47` style — letter for color, raw line count for context.
- **Tests:** size classifier matrix (boundary conditions); unit tests for the size-tag DOM helper.
- **AC:** sizes computed off `state.additions + state.deletions` already returned by `/pulls/:n`; no new fetch.

#### QM-502 — Comments indicator + jump
- **Files:** `lib/qm-row-badges.js`, `theme.css`.
- **Behavior:** show the `comments` field as a chat-bubble + count. Anchor links to `/<owner>/<repo>/pull/<n>#issue-comment-area`. Hidden when count is 0.
- **Tests:** href format; hidden-when-zero behavior.
- **AC:** keyboard-reachable; `aria-label="3 comments — jump"`.

#### QM-503 — Ready-to-merge highlight
- **Files:** `lib/qm-row-badges.js`, `theme.css`.
- **Behavior:** add `data-qm-ready="true"` to the row container when `state.mergeable_state === 'clean'` AND viewer-approved (or no review required by branch protection — a proxy: `state.mergeable === true && state.behind_by === 0`). CSS draws a subtle 2px green left-edge.
- **Tests:** flag-on / flag-off matrix.
- **AC:** doesn't conflict with the existing repo-default highlight; cascade order documented.

### F11.2 — Quick-filter bar (Track B)

#### QM-504 — Filter bar mount + chips UI
- **Files:** `lib/qm-filter-bar.js`, `theme.css`, `content.js` (mount call once `.repository-content` is detected).
- **Behavior:** four chips: **Mine / Ready / Stale / Small**. Toggleable; persisted in `chrome.storage.sync.qm_filters` as a `{ [name]: true }` map. A chip is active = highlighted with `--qm-primary-soft`.
- **Tests:** chip render; toggle persistence; multi-active state.
- **AC:** chips match the `ui_ux.md` §Filters mock; mount idempotent across pjax / turbo.

#### QM-505 — `Mine` filter
- **Files:** `lib/qm-filter-bar.js`.
- **Behavior:** show only rows where `pr.author === viewer.login`. Reuses the cached viewer fetched by Epic 10 (`prPageState.viewer`).
- **Tests:** unit — predicate; integration — row hide/show in fixture DOM.

#### QM-506 — `Ready` filter
- **Behavior:** show only rows where `state.mergeable_state === 'clean'`. Same flag QM-503 keys off.
- **Tests:** predicate matrix.

#### QM-507 — `Stale` filter
- **Behavior:** show only rows where `lib/stale-pr.js#isStale(pr) === true`. Already-shipped library does the threshold work.
- **Tests:** wire-up only — stale logic is its own test surface.

#### QM-508 — `Small` filter
- **Behavior:** show only rows where size class ∈ {XS, S} (≤50 lines changed).
- **Tests:** predicate matrix off the QM-501 classifier.

### F11.3 — Noise toggles (Track C)

#### QM-509 — Dependabot toggle
- **Files:** `lib/qm-filter-bar.js` (extension; not a separate module).
- **Behavior:** "Hide dependabot" chip — when active, rows where `pr.author === 'dependabot[bot]'` are filtered out. OR'd with the active include-filters (Mine etc.) — exclusion runs after inclusion.
- **Tests:** include + exclude composition matrix.

#### QM-510 — Drafts toggle
- **Behavior:** "Hide drafts" chip — filters out `pr.draft === true`.
- **Tests:** predicate matrix.

- **AC:** ≥95% line coverage on all three modules.

#### QM-512 — Playwright e2e — filter bar

## Risk register

| Risk | Likelihood | Mitigation |
|------|------------|------------|
| Combined-status fetch per row blows the rate budget | Med | Reuse the existing `state.cache` keyed by `${owner}/${repo}#${num}`; piggyback the status fetch onto the existing `fetchPrState` round-trip via the GraphQL `prs(...) { commits(last:1) { ... statusCheckRollup }}` shape — single network call per row. |
| Filter logic interaction matrix grows quadratically | Med | Predicates are pure functions composed by `Array.prototype.every`; QM-511 covers the include × exclude × multi-active combinations explicitly. |
| Filter persistence races with pjax remounts | Low | Single `qm_filters` read at mount; subsequent toggles write through. The existing `chrome.storage.onChanged` listener in `content.js` rebroadcasts. |
| Badge injection drives row-injection p95 above the perf gate | Med | Badges build is synchronous DOM (no fetch in the badge code path itself — data is already on `state.cache`). Add a perf assertion to `test/e2e/perf/injection-timing.spec.ts` to gate. |
| Dependabot author detection misses GitHub Actions, Renovate, etc. | Low | Match against a configurable allow-list `qm_noise_authors` defaulting to `['dependabot[bot]', 'renovate[bot]', 'github-actions[bot]']`; users can extend. |

## Code-quality checklist

- [ ] All new selectors + fetches centralized in `lib/hosts/github/`.
- [ ] Badge module is pure DOM — no `fetch()`, no `chrome.*`. `content.js` injects state.
- [ ] Filter bar module reads / writes `chrome.storage.sync.qm_filters` once on mount + on toggle.
- [ ] Idempotency: badge / filter-bar re-mount is a no-op when their flag attrs already exist.
- [ ] Soft-nav resilient: hooked into the existing `MutationObserver` + `pjax:end` / `turbo:render`.
- [ ] A11y: chips have `role="button"`, `aria-pressed`; badges have `aria-label`.
- [ ] Dark-mode tokens honored — uses `--qm-success / -danger / -warn` etc; no hard-coded hex.
- [ ] Coverage gate ≥ 95% lines.

## Workflow decision: 4 PRs, squash-merge each

**Recommendation: split by track (A, B, C, D), squash-merge each.**

Rationale:
1. Tracks A and B touch disjoint files. Splitting reduces review surface per PR.
2. Track C is a small extension of B's storage shape; lands after B.
3. Track D's tests bundle alongside their implementation track — no separate "tests-only" PR.
4. A single sub-agent fan-out (per `dispatching-parallel-agents`) is viable for A + B if the user wants speed; default is sequential for clarity.

## Acceptance gate

Before `gh pr ready` on each track:
1. Local green: `npm test` (≥95% lines on touched modules).
2. `npx playwright test` green including refreshed baselines.
3. Perf gate: row-injection p95 < 50 ms (existing perf spec).
4. Manual smoke: real github.com `/pulls` against a multi-repo workspace; verify badges render; chips toggle; persistence across reload.
5. `codex-adversarial-gate` clean (Track A only — touches new fetch surface).
