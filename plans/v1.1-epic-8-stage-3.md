## Execution mode: §3a Direct, single track

Reasoning:
- **File overlap is heavy.** popup.html/popup.js is touched by QM-211, QM-215, QM-219. options.html/options.js by QM-212, QM-213, QM-218, QM-219. content.js by QM-209, QM-214, QM-219. Three subagents would collide on every shared file.
- **Soft sequential deps.** QM-213 hangs off QM-212 (needs the side-nav). QM-218 needs QM-219's storage plumbing landed first or it's a refactor immediately. QM-215 reads popup state from QM-211. QM-220 must run *last* by definition (snapshots all preceding surfaces).
- The dispatch-or-direct heuristic says: ≤3 tracks with shared files → direct.

Single feature branch, one PR, one squash-merge.

## Implementation order

Within the single branch, land in this internal order so nothing regresses mid-wave:

1. **QM-219 storage plumbing** — `lib/qm-visual-prefs.js` reader/writer + `chrome.storage.onChanged` subscription helper. Unit tests for default fallback + cross-surface broadcast. **No UI yet.**
2. **QM-209 shortcut hint** — small, isolated; warm-up before bigger refactors. Reads `state.shortcutMode` already plumbed in stage 2.
3. **QM-211 popup rebuild** — replace popup.html structure; popup.js re-uses existing fetch/render logic, just re-templated. Keep all existing event handlers wired.
4. **QM-215 onboarding** — gate on `!clientId && !token && !onboardingDismissed`; renders inside popup shell from QM-211. New `lib/qm-onboarding.js` for the card markup + dismiss logic. Closes QM-174.
5. **QM-212 options side-nav** — restructure options.html into nav + pane layout. Each existing options.js pane becomes a `<section data-pane="…">`. JS toggles `[hidden]`.
6. **QM-213 About pane** — append section, mostly static markup + version pulled from `manifest.json`.
7. **QM-218 Tweaks panel** — new "Appearance" section in options. Writes via QM-219 helper.
8. **QM-214 bulk bar redesign** — replace `ensureBulkBar` HTML in content.js. Idle / mid-flight states. Uses `QM_TOAST` for completion feedback (already shipped stage 1).
9. **QM-220 visual snapshot refresh** — run `npx playwright test --update-snapshots` for popup/options/modals; eyeball each diff per `docs/snapshot-update.md`; commit baselines.

  - Open options → side-nav switches panes; About shows version; Tweaks live-previews accent/density/font.

  - Trigger bulk merge → new dark-pill bar with per-row queued/running/ok status.
  - Hover row in `prs/` page → "▶ press S to squash" hint when shortcut-mode is active.
- `prefers-color-scheme: dark` still flips correctly across all redesigned surfaces.

## Risks

- **Snapshot churn.** QM-220 will produce a large diff of PNG baselines. Mitigate: review locally, eyeball each, commit baselines as a separate logical commit within the branch so reviewers can skip the PNG byte-noise.
- **Options side-nav refactor risk.** options.js historically renders panes inline; moving to `[data-pane]` toggling could miss an event handler. Mitigate: keep existing handlers, only restructure markup; verify each pane functionally on smoke.
- **Bulk-bar state machine.** Idle ↔ mid-flight ↔ paused has more states than current code. Mitigate: extract small reducer in content.js (or `lib/qm-bulk-bar.js` if it grows past ~80 lines).
