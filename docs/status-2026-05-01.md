## 4. Risks / known issues

- **Visual baseline drift is permanent CI red** until QM-220 runs. The 11 failing specs all share the `writing actual` signature. Per `~/.claude/CLAUDE.md` "Playwright visual baselines are per-platform; pull from CI artifacts" — running this locally on Darwin won't produce baselines that match CI's Ubuntu pixels. Capture from CI artifacts.
- **Manifest / package.json still on 0.4.0** despite Epic 8/10/11 having all merged. Either the version bumping cadence is intentional ("bump on store submission") or this is a stale artifact — worth confirming before the next release.
- **`lib/qm-filters.js` retains `hideDrafts` logic**, and the filter bar wires it (`data-qm-filter="hideDrafts"`), so this is **not** dead code — it's the live v1.3 noise toggle (PR #51). The earlier "removed feature" reference was about `qm-cancel-watch-btn` / `qm-queue-badge` only.
- **Cancel-watch button removal is clean**: dead CSS pulled from `theme.css` and `styles.css`; doc comment in `content.js` updated. `ROADMAP.md` line 104 still mentions "cancel-watch" as part of the F2.2 description — historical / acceptable, but flag if you want roadmap to reflect removal.
- **Epic 9 partial landing risk**: HostAdapter scaffold (PR #39) introduces an abstraction that has only one consumer (GitHub) until F9.1 lands. This is the "premature interface" pattern — keep the GitLab port on the critical path or revert the scaffold.
- **Sentry is half-shipped**: consent UI + sanitizer + tests merged (PR #38), but SDK not vendored (QM-169). Telemetry calls are no-ops in production until the package script runs `vendor:sentry`.
- **Branch protection unconfigured** despite repeated CI-discipline guidance in CLAUDE.md and the dedicated launch follow-up row. Trivial fix; high embarrassment cost if a force-push lands.
- **Coverage gate exists** (`vitest run --coverage`) but was not executed in this audit — no asserted coverage number for this report.
- **No test gap detected** for the v1.1 row-widget refine pass beyond visual snapshots. `test/qm-brand.test.js`, `qm-visual-prefs.test.js`, `qm-theme.test.js`, `qm-size-classify.test.js`, `qm-filters.test.js` (34 tests), `qm-auto-rebase-toggle.test.js` are all green and cover the new lib modules.
- **`.playwright-user-data/` is committed-or-tracked-shaped** (showed up in `find` walk). If this is gitignored already, ignore; if not, it's hundreds of MB of Chrome profile cruft that should be in `.gitignore`.
